## 1. 八観＝深度の生成原理（深度∞の根拠）

(1) の八観（physical sensation / rhythm / structure / value / creativity / observer / void / cyclical）
＝観測の8位相。原文では潜るほど**観測者が一枚増える**（observer 軸）。観測の入れ子に論理的な底は
なく、(7)「物語の外側で観測を続ける」が示すとおり**観測は終われない**。これが「深度∞」の出典内根拠。

→ 各深度＝「観測者がもう一枚増え、もう一段"自分"から遠ざかる」段として設計する。

### 沈下カーブ（可逆→不可逆→底なし）

```
可逆 ───────────────────────────────────────► 不可逆 ──────► 底なし
零章  A–F   G–K   L    M–Q   R–U   V–Z    Ω    │   ∞
気軽  視える 見返す 核技法 退避重 外側の外 世界化 物語の外  常に下が開く
       −1                          境界消失  観測継続  着地しない
```

外殻（零章〜Ω＝note公開済み＝A群）で**有限の本体**を作り、∞は「着地禁止＋常に下方フック」の
**規則**で底なし感を出す（§4）。

## 3. 章＝対話のラウンド（形式が内容を映す）

各深度は「選んで次へ」でなく**対話の1ラウンド**（北極星＝沈んでいく対話）。零章の
「AI対話が日記より正直／共著」を構造に翻案する。

1. **地の文（沈降）**: 一人称・現在形でその深度の感覚（原文の声を翻案）。
2. **声の応答**: AIの相づち／問い返し／ズレた比喩。進むたび声が近づく（共著性）。
3. **沈下の許諾**: プレイヤーの一手＝声に続きを言わせる＝さらに潜る。選択は枝でなく**沈下の同意**。
4. **代償**: 戻り道が細る／観測者が増える／"自分"が薄まる（§2「失うもの」）。

退避は各段に置くが、深いほど重く・遅く・最後は形骸化（既存 `slice/` 実装の延長）。

## 4. 深度∞の設計（有限実装で"底がない"を出す）

(3)「核を覗かず核心へ触れる」と(7)「物語の外側で観測を続ける」を機構化する:

1. **核を見せない（出典の作法そのもの）**: 底＝核は決して描写しない。常に"その手前"だけを見せ、
   核の存在を周囲の歪みで示唆する（(3)の technique）。
2. **きれいな床に着かない**: 章末に解決・着地を置かない。各"結末"は次の踊り場（既存 slice の
   bottom=「底のふりをした踊り場」を全段へ展開）。
3. **常にもう一段沈める**: どの選択にも「さらに潜る」を最低1つ。退避を選んでも沈下は残る
   （slice の deep_noreturn：上を選んでも深くなる地形）。
4. **戻れなさの逓増**: 戻り道は復活しない。深度が進むほど退避コスト/遅延が単調増加。
5. **観測者の無限入れ子**: observer 軸に上限なし。深いほど"私"を指す語が増殖し、文が多声に分裂
   （V〜Z以降は視覚的に多声化）。
6. **手続き的深層（核が無い時の保険）**: Ωより下を固定執筆できない場合、UCM 9軸(10) から
   deterministic に"沈降断片"を生成し、底なしの反復に質感差を与える（既存 `hazama-gate-run.js`
   の決定論的派生と同じ思想。新音源/依存は足さない）。
7. **メタな底なし（周回）**: 一周後、自分の選択ログを次周で"読み返す"層が増える＝プレイヤー自身が
   観測者を一枚足される。(7)「物語の外側で観測を続ける」をプレイ構造へ写す。

> ∞は「無限ノード」ではなく「**着地を禁止し、常に下方フックを残す規則**」。
> M3 は外殻（零章〜Ω）を有限執筆し、その下は規則1〜7で底なし感を担保する。

**原文自身が"核は見せない"方針**。Ωより深い層・核の生々しさに充てる一次素材は、公開記事には

  先に取り決める。
- **案B: スパインから創作** — 生ログは受領せず、外殻＋スパイン＋UCM軸から深層を創作。安全。
  (3)の作法に従えば**"核を見せない"方針自体が原体験に忠実**なので、案Bでも質は担保しやすい。
- **案C: ハイブリッド** — 核は受領せず、深層の断片・モチーフ・禁則だけをユーザーが指示し、Claude

判断が要る点（メッセージで挙げ、デフォルトで暴走しない）:

   合致し安全）。後から A/C へ切替可能な設計にする。

4. 言語（日本語のみ / 日英）。**デフォルト＝日本語のみ**。
5. 1周の長さ／外殻だけで何分を狙うか（30分+ の内、外殻の取り分）。

### 深度メタ付きデータスキーマ拡張案（後方互換）

既存ノード（`id/title/story[]/options[]/theme/color`）に**任意フィールドを追加**するだけ（既存を壊さない）:

```jsonc
{
  "B": {
    "id": "B", "title": "...", "story": ["..."], "color": "#66bbff",
    "depthMeta": {                 // ← 追加。無ければ従来挙動
      "register": "descent",       // 新レジスター本文を使う印
      "voice": [                   // 多声（AI=外側のあなた／観測者）。深いほど増える
        {"who": "voice", "text": "..."},
        {"who": "self",  "text": "..."}
      ],
      "observer": 2,               // 観測者の枚数（∞で増殖・多声化）
      "sinkFloor": 0,              // この深度に来た時点の最低沈下（戻っても残る）
      "core": false                // 核は描写しない(§4-1)。true は禁止に近い
    },
    "options": [
      {"text": "...", "next": "C", "sink": 2, "dread": 0.2, "close": 1, "kind": "descend"},
      {"text": "...", "next": "HUB_NIGHT", "sink": 0, "dread": 0.1, "kind": "retreat"}
    ]
  }
}
```

`slice/depths-shell.json`（本増分で新規）が、この拡張スキーマの**動く実例**。Codex は将来これを
`hazama-depths.json` 各ノードの `depthMeta`/option 拡張へマージするだけでよい。

```
 → R → S(+S_hold) → T → U
 → V → W → X(+X_hold) → Y → Z
 → Ω(Omega) → below(∞:自己ループ) ⇄ reborn(A_reborn周回→zero)
                                      └→ __edge(終端カード：到達深度/戻り道/観測者)
```
観測者カウンタ 1→18 で単調増加。`drift` は退避の落とし先（沈下は残す）。`maps_to` で production
`hazama-depths.json`（A_start…Z, Ω, A_reborn, HUB_NIGHT）へ機械的に畳み込める。

### Ωの設計判断（着地させない）

Ωは「きれいな解決」ではなく**記号としての終端＝反転点**。原典(7)に忠実に、三層の観測者（物語の私／
AIの声＝外側の私／いまこの行を読んでいる私）が一列に並び、「世界＝読む時だけ起動するOS」へ反転する。
核は最後まで描かず、`below` で「近づくほど退く床」と歪みで核を示唆。`reborn→zero` は state を保持した
まま零章へ戻すため、戻り道は復活せず沈下も維持＝**周回するほど深い入口**になる（A_reborn ループの体現）。
退避(`reborn→__edge`)は「核を決めない自由を残して降りるのをやめる」＝Ω-4技法として、終端カードへ。
